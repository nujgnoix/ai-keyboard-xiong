# DeepSeek中转服务 - 技术交接文档

## 📋 项目简介

DeepSeek中转服务是一个纯Java实现的自然语言转键盘指令的转换模块，将用户自然语言输入转换为结构化键盘操作（按键组合或剪贴板粘贴）。

**核心能力：**
- 用户输入：自然语言（如"把前三个字加粗"）
- DeepSeek输出：结构化Actions数组
- 客户端：将Actions交给现有的蓝牙HID键盘逻辑

**版本：v3.0（含三期功能）**

## 🚀 三期功能概览

### 第一期：剪切板上下文

**功能说明：** 在调用AI前，先获取电脑端文本内容，让AI能基于现有文本执行操作。

**工作流程：**
```
用户输入自然语言
    ↓
发送 Ctrl+A → Ctrl+C（全选+复制）
    ↓
Android端读取剪切板内容
    ↓
把剪切板内容加入上下文 → 调用DeepSeek
    ↓
生成Action队列 → 执行
```

**使用场景：**
- "把前面的'Apple'替换为'Google'" → 先看当前文本
- "给第三段加粗" → 先了解段落结构
- "在结尾加上'谢谢'" → 先看现有结尾

---

### 第二期：多轮对话与记忆管理

**功能说明：** 记住用户的历史操作和偏好，提供更智能的服务。

**记忆架构：**
```
MemoryManager
    ├─ ShortTermMemory（短期记忆）
    │   ├─ 最近20条对话记录
    │   ├─ 剪切板快照历史
    │   └─ 执行结果记录
    │
    └─ LongTermMemory（长期记忆）
        ├─ 对话摘要（最多50条）
        ├─ 用户偏好（字号、字体等）
        └─ 任务模板
```

**关键参数：**
| 参数 | 值 | 说明 |
|------|---|------|
| SHORT_TERM_CAPACITY | 20 | 短期记忆容量 |
| LONG_TERM_CAPACITY | 50 | 长期记忆容量 |
| SUMMARY_THRESHOLD | 10 | 触发总结阈值 |

---

### 第三期：技能系统与自学习

**功能说明：** 从用户操作中学习新技能，自动记录用户偏好。

**核心组件：**
| 组件 | 说明 |
|------|------|
| SkillManager | 技能注册、匹配、学习 |
| Skill | 技能数据模型（触发词+动作序列） |
| UserPreferenceManager | 用户偏好学习与提取 |

**自学习流程：**
```
用户执行操作
    ↓
SkillManager.learnFromActions()
    ↓
提取关键词 → 检测相似技能 → 创建新技能
    ↓
下次用户输入匹配关键词 → 自动推荐技能
```

## 💻 代码集成示例

### 基础使用（单轮对话）

```java
DeepSeekClient client = new DeepSeekClient(
    BuildConfig.DEEPSEEK_API_KEY,
    Platform.WINDOWS
);

List<Action> actions = client.execute("把前三个字加粗");
```

### 带剪切板上下文（第一期）

```java
// 先获取剪切板内容
String clipboardContent = captureClipboardFromPC();

// 带上下文调用
List<Action> actions = client.execute("把Apple改成Google", clipboardContent);
```

### 带记忆管理（第二期）

```java
MemoryManager memoryManager = new MemoryManager();

// 记录用户操作
ActionRecord record = new ActionRecord("把前三个字加粗", clipboardContent);
record.setActions(actions);
memoryManager.addRecord(record);

// 获取上下文提示
String contextPrompt = memoryManager.injectContextPrompt();
```

### 技能学习（第三期）

```java
SkillManager skillManager = new SkillManager();

// 从用户操作学习新技能
Skill newSkill = skillManager.learnFromActions("加粗标题", actions);
skillManager.registerSkill(newSkill);

// 匹配已有技能
Skill matched = skillManager.findMatchingSkill("帮我加粗一下标题");
if (matched != null) {
    // 直接使用技能的动作序列
    executeActions(matched.getActions());
}
```

### 完整集成示例

```java
public class AIKeyboardService {

    private DeepSeekClient client;
    private MemoryManager memoryManager;
    private SkillManager skillManager;
    private UserPreferenceManager preferenceManager;

    public void initialize(String apiKey, Platform platform) {
        this.client = new DeepSeekClient(apiKey, platform);
        this.memoryManager = new MemoryManager();
        this.skillManager = new SkillManager();
        this.preferenceManager = new UserPreferenceManager();
    }

    public List<Action> processUserInput(String userInput) {
        // 1. 先检查是否有匹配的技能
        Skill matchedSkill = skillManager.findMatchingSkill(userInput);
        if (matchedSkill != null) {
            matchedSkill.incrementUsage();
            return matchedSkill.getActions();
        }

        // 2. 获取剪切板上下文
        String clipboardContent = captureClipboardFromPC();

        // 3. 获取用户偏好提示
        String preferencePrompt = preferenceManager.getPreferencesPrompt();

        // 4. 调用AI
        List<Action> actions = client.execute(userInput, clipboardContent);

        // 5. 记录到记忆系统
        ActionRecord record = new ActionRecord(userInput, clipboardContent);
        record.setActions(actions);
        memoryManager.addRecord(record);

        // 6. 尝试学习新技能
        skillManager.learnFromActions(userInput, actions);

        return actions;
    }

    private String captureClipboardFromPC() {
        // 发送 Ctrl+A → Ctrl-C
        sendHidKeys("ctrl", "a");
        sendHidKeys("ctrl", "c");

        // 读取剪切板
        return readClipboard();
    }
}
```

## 📊 数据结构

### Action对象结构

```java
public class Action {
    private ActionType type;        // KEY_COMBO 或 CLIPBOARD_PASTE
    private List<String> keys;      // 按键组合（仅KEY_COMBO有）
    private String text;            // 粘贴文本（仅CLIPBOARD_PASTE有）
    private String description;     // 操作描述
}
```

### Skill对象结构

```java
public class Skill {
    private String id;                  // 技能ID
    private String name;                // 技能名称
    private List<String> triggerKeywords; // 触发关键词
    private List<Action> actions;       // 动作序列
    private int usageCount;             // 使用次数
    private long lastUsedAt;            // 最后使用时间
}
```

## 🔍 故障排查

### 常见问题

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| API Key错误 | Key无效或过期 | 检查DeepSeek控制台并更新 |
| JSON解析失败 | DeepSeek返回格式不对 | 查看Prompt规则 |
| 剪切板内容为空 | 复制操作未完成 | 增加延迟等待时间 |
| 技能匹配不准 | 关键词不够准确 | 增加更多触发关键词 |
| 记忆占用过大 | 长期使用积累 | 调用clearOldMemories()清理 |

## ⚠️ 注意事项

1. **权限要求：** 需要 `INTERNET` 权限

```xml
<uses-permission android:name="android.permission.INTERNET" />
```

2. **线程安全：** 调用API必须在非UI线程执行

3. **剪切板敏感数据：** 避免在密码输入场景使用剪切板捕获功能

4. **记忆容量：** 定期清理长期记忆，避免占用过多存储空间
